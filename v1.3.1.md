# v1.3.1

Bug-fix release: concurrent Yaw windows no longer collide on each other's `.claude/tasks/` state.

## Fixes

- **Concurrent sessions don't block each other anymore.** Each Yaw overlay now gets a private `tasks/` directory instead of sharing `~/.claude/tasks/`. The old behavior caused "Access is denied" errors when one Yaw session tried to inspect a directory held open by another session's Claude Code process -- most visible when running multiple Yaw terminals at once.
- **Leaked session dirs get cleaned up automatically.** On app start, Yaw sweeps `~/.claude/tasks/<uuid>/` directories whose owning Claude Code process is gone and whose mtime is older than 24h. Live sessions are detected via their `.lock` handle and left alone.

## New tools

- `scripts/gc-stale-tasks.mjs` -- manually GC leaked task dirs. Supports `--dry-run` and `--ttl=24h` overrides.
- `scripts/yaw-doctor.mjs` -- diagnoses blocked-dir symptoms by surfacing which Claude Code PIDs are live and which shell identity likely owns the lock.

Both are opt-in; the automatic in-app sweep handles the common case without you running anything.
